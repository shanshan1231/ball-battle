# AI小球观战对战游戏 - 项目规范

## 1. 项目概述
- **项目名称**: Ball Battle Arena (小球对战擂台)
- **类型**: 单人单机AI自动对战游戏
- **核心功能**: 玩家选择两个AI单位，观看它们在擂台上自动战斗
- **目标用户**: 休闲游戏玩家，喜欢观战类游戏的用户

## 2. 技术架构

### 后端 (Node.js + Express + WebSocket)
- Express 处理静态文件服务
- WebSocket (ws库) 处理实时战斗数据同步
- 游戏逻辑全部在后端计算，保证公平性

### 前端 (HTML + CSS + 原生JS)
- 三个页面: 选角页、战斗页、结算页
- Canvas 渲染战斗画面
- WebSocket 客户端通信

## 3. 核心玩法设定

### 3.1 游戏流程
1. **选角页面**: 玩家从5个职业中选择【对战单位A】和【对战单位B】
2. **战斗页面**: 两个AI小球在擂台上自动对战
3. **结算页面**: 显示胜负结果，提供重新挑战按钮

### 3.2 战斗场地
- 正方形封闭擂台 (800x800 像素)
- 深色背景 (#1a1a2e)
- 擂台边界为实体墙，小球碰撞后反弹

## 4. 小球职业详细设计

- **外观**: 红色小球，刀光特效

### 4.3 盾牌手 (Shield Bearer)

## 5. 战斗数值与状态机制

### 5.1 血量系统
- 初始血量根据职业固定
- 血条颜色: 绿(>50%) → 黄(25%-50%) → 红(<25%)
- 血量归零判定死亡

### 5.2 状态效果
- **减速**: 被攻击后移速降低50%，持续500ms
- **冷却**: 各职业独立攻击冷却

### 5.3 碰撞系统
- 小球之间物理碰撞挤压
- 小球与墙壁碰撞反弹

## 7. 页面设计

### 7.1 选角页面 (index.html)
- 5个职业卡片展示

## 8. WebSocket 通信协议

### 8.1 消息类型
- `start_game`: 开始游戏
- `game_state`: 游戏状态更新 (60fps)
- `attack`: 攻击事件
- `hit`: 命中事件
- `death`: 死亡事件
- `game_over`: 游戏结束
- `restart`: 重新开始

### 8.2 游戏状态结构
```javascript
{
  ballA: { x, y, vx, vy, hp, maxHp, class, attackCooldown },
  ballB: { x, y, vx, vy, hp, maxHp, class, attackCooldown },
  projectiles: [], // 投射物数组
  effects: []      // 特效数组
}
```
